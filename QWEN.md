## Qwen Added Memories
- Project: Tether - mobile web terminal controller with tmux session mirroring. Latest commit: 2c70d7c on main branch. Key feature added: bidirectional tmux mirroring - existing tmux sessions appear in web app as panels, input/output syncs both ways. Architecture: TmuxManager (src/tmux_manager.rs) discovers tmux panes via `tmux list-panes`, captures content via `tmux capture-pane`, streams real-time output via `tmux pipe-pane` to FIFOs, sends input via `tmux send-keys`. Frontend auto-discovers tmux sessions on load via GET /api/tmux/sessions, creates panels for each pane with WebSocket connection using `?tmux_pane=` param. tmux is NOT installed yet on this machine - user needs to run `sudo apt install tmux`. Build: `cargo build --release`, binary at `target/release/tether`. Startup: `./run.sh [password] [port]` starts tether + ngrok. All endpoints tested and working (login 200, CSS 200, JS 200, auth gate 401, tmux API returns 503 when tmux not installed, terminal creation works).
